# <a name="cmfcvisualmanager-class"></a>CMFCVisualManager 类

提供在全局级别更改应用程序外观的支持。 `CMFCVisualManager` 类与说明如何绘制应用程序中 GUI 控件的类一起使用，且使用一致样式。 这些其他类称为视觉管理器，从 `CMFCBaseVisualManager`继承而来。

有关详细信息，请参阅位于 Visual Studio 安装的**VC\\\\atlmfc src\\mfc**文件夹中的源代码。

## <a name="syntax"></a>语法

```
class CMFCVisualManager : public CMFCBaseVisualManager
```

|----------|-----------------|
|`CMFCVisualManager::CMFCVisualManager`|默认构造函数。|

## <a name="remarks"></a>备注

由于`CMFCVisualManager`类控制应用程序的 GUI，因此每个应用程序可以具有 一个`CMFCVisualManager`实例或从 派生的`CMFCVisualManager`类的一个实例。 应用程序也可以在没有 。 `CMFCVisualManager` 使用静态方法`GetInstance`获取指向当前`CMFCVisualManager`派生对象的指针。

要更改应用程序的外观，必须使用其他类，这些类提供绘制应用程序所有可视元素的方法。 这些类的一些示例包括[CMFCVisualManagerOfficeXP 类](../../mfc/reference/cmfcvisualmanagerofficexp-class.md)[、CMFCVisualManagerOffice2003 类](../../mfc/reference/cmfcvisualmanageroffice2003-class.md)和[CMFCVisualManagerOffice2007 类](../../mfc/reference/cmfcvisualmanageroffice2007-class.md)。 如果要更改应用程序的外观，请将这些可视化管理器之一传递到 方法`SetDefaultManager`中。 有关演示应用程序如何模拟 Microsoft Office 2003 外观的示例，请参阅[CMFCVisualManagerOffice2003 类](../../mfc/reference/cmfcvisualmanageroffice2003-class.md)。

所有绘图方法都是虚拟的。 这使您能够为应用程序的 GUI 创建自定义可视样式。 如果要创建自己的视觉样式，请从其中一个可视化管理器类派生类，并重写要更改的绘图方法。

## <a name="example"></a>示例

此示例演示如何实例化标准和自定义`CMFCVisualManager`对象。

```cpp
void CMFCSkinsApp::SetSkin (int iIndex)
{   // destroy the current visual manager
    if (CMFCVisualManager::GetInstance () != NULL)
    {
        delete CMFCVisualManager::GetInstance ();
    }
    switch (iIndex)
    {
    case 0:
        CMFCVisualManager::GetInstance ();
        // create the standard visual manager
        break;
    case 1:
        new CMyVisualManager ();
        // create the first custom visual manager
        break;
    case 2:
        new CMacStyle ();
        // create the second custom visual manager
        break;
    }
    // access the manager and set it properly
    CMFCVisualManager::GetInstance ()->SetLook2000 ();
    CMFCVisualManager::GetInstance ()->RedrawAll ();
}
```
