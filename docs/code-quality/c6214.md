# <a name="c6214"></a>C6214

> 警告 C6214：语义不同的整数类型之间的强制转换： HRESULT 到布尔类型

此警告表明 `HRESULT` 被强制转换为布尔类型。 `HRESULT` 的成功值（`S_OK`）等于0。 但是，0指示布尔类型的失败。 将 `HRESULT` 强制转换为布尔类型，然后在测试表达式中使用它会产生不正确的结果。 有时，如果 `HRESULT` 存储在布尔变量中，则会出现此警告。 使用布尔变量测试 `HRESULT` 成功或失败的任何比较都可能导致不正确的结果。

## <a name="example"></a>示例

下面的代码将生成此警告：

```cpp
#include <windows.h>

BOOL f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;
  hr = CoGetMalloc(1, &pMalloc);
  if ((BOOL)hr) // warning C6214
  {
    // success code ...
    return TRUE;
  }
  else
  {
    // failure code ...
    return FALSE;
  }
}
```

若要更正此警告，请使用以下代码：

```cpp
#include <windows.h>

BOOL f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (SUCCEEDED(hr))
  {
    // success code ...
    return TRUE;
  }
  else
  {
    // failure code ...
    return FALSE;
  }
}
```

对于此警告，`SCODE` 类型等效于 `HRESULT`。

通常，应使用 `SUCCEEDED` 或 `FAILED` 宏来测试 `HRESULT`的值。
