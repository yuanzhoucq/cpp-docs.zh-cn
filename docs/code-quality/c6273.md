# <a name="c6273"></a>C6273

> 警告 C6273：作为参数传递的非整数 \<数字 > 在对 \<函数的调用中需要整数时 >：如果传递的是指针值，应使用% p

此警告表明，格式字符串指定一个整数，例如，`printf` 的 `%d`、长度或优先级规范，但非整数，如 `float`、string 或 `struct` 将作为参数传递。 此缺陷可能会导致不正确的输出。

## <a name="example"></a>示例

下面的代码将生成此警告，因为需要整数而不是 `float` 来 `sprintf`函数：

```cpp
#include <stdio.h>
#include <string.h>

void f_defective()
{
  char buff[50];
  float f=1.5;

  sprintf(buff, "%d",f);
}
```

下面的代码使用整数 cast 来更正此警告：

```cpp
#include <stdio.h>
#include <string.h>

void f_corrected()
{
  char buff[50];
  float f=1.5;

  sprintf(buff,"%d",(int)f);
}
```

下面的代码使用安全的字符串操作函数（`sprintf_s`）来更正此警告：

```cpp
#include <stdio.h>
#include <string.h>

void f_safe()
{
  char buff[50];
  float f=1.5;

  sprintf_s(buff,50,"%d",(int)f);
}
```

此警告不适用于 Windows 9x 和 Windows NT 版本4，因为在这些平台上不支持% p。
