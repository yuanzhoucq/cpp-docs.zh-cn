# <a name="c6286"></a>C6286

> 警告 C6286：（ \<non-zero constant> &#124;&#124; \<expression> ）始终为非零常量。 \<expression>永远不会被计算，可能会有副作用

此警告意味着在测试上下文中发生的逻辑或运算的左侧检测到非零常量。 生成的表达式的计算结果始终为 TRUE。 此外，表达式的右侧看似有副作用，它们会丢失。

此警告表明您可能需要仔细检查表达式的右侧，以确保正确功能所需的任何副作用不会丢失。

`(!0 || <expression>`）构造通常用于强制执行受控块。

## <a name="example"></a>示例

下面的代码将生成此警告：

```cpp
#include <stdlib.h>
#include <stdio.h>
#define INPUT_TYPE 1

int test();

void f()
{
  if (INPUT_TYPE || test())
  {
    puts("INPUT_TYPE == 1, expression not evaluated");
    // code...
  }
  else
  {
    puts("INPUT_TYPE == 0. Call to test() returned 0");
    // code...
  }
}
```

下面的代码通过将 **`if`** 语句分解为两个单独的部分来显示一种可能的解决方案：

```cpp
#include <stdlib.h>
#include <stdio.h>
#define INPUT_TYPE 1

int test();

void f()
{
  int i;
  if (INPUT_TYPE)
  {
    i = test();
    // code...
  }
  else
  {
    puts("INPUT_TYPE false");
    // code...
  }
}
```
